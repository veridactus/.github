# Security Policy

## Supported Versions

| Version | Status |
|:--------|:-------|
| `0.2.x` | ✅ Active development |
| `0.0.x` | ⚠️ Security fixes only |

## Reporting a Vulnerability

**⚠️ Do not report security vulnerabilities via public GitHub Issues.**

Instead, send an encrypted email to:

📧 **`security@veridactus.ai`**

Include the following information:
- Vulnerability type (e.g., authentication bypass, data leak, DoS)
- Affected version/component
- Minimal, verifiable reproduction steps
- Potential impact assessment

We will acknowledge receipt within **48 hours** and provide a preliminary fix timeline within **7 business days**.

## Disclosure Policy

- Once a fix is available, we coordinate a **30-day public disclosure window**.
- Reporters who follow responsible disclosure will be acknowledged in the security advisories.
- Critical vulnerabilities may warrant an out-of-band release; the TSC decides on a case-by-case basis.
